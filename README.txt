綜合QR Code

關於在QR Code的編、解碼流程上，增加一些額外機制的實作。

現有的QR Code有兩個問題
1.QR Code缺乏內容驗證機制
    提供給使用者的QR Code，無法確保資訊來源的正確性，可能遭惡意攻擊者竄改內容，甚至偽造加入釣魚台網站連結。

2.QR Code提供資訊上的限制
    最大儲存量只有3KB，造成資料格式應用有限，只能儲存文字、符號，必須透過網路的方式，才能提供更多的資訊給使用者。但是這樣受限於訊號的強弱與使用者的網路流量，在無網路服務的地方，不易使用。

1.Simple QR Code
    基本的QR Code解碼器，解碼後呈現的是一般的文字，或網址。

2.Hash QR Code
    將明文進行雜湊運算，與產生的雜湊值一同編於QR Code內。解碼時會先驗證雜湊值，確認內容無遺失或改變後，再呈現給使用者。可用於一般的應用環境。

3.HMAC QR Code
    雜湊訊息確認碼，將雜湊值與系統隨機產生的金鑰進行運算，產生HMAC驗證值，並與明文一同編於QR Code內，而雜湊值與金鑰存於資料庫中。解碼時會先計算雜湊值，並於資料庫中取出對應的金鑰，再計算出HMAC驗證值，與QR Code內的進行比對，確認內容無誤後再呈現給使用者。由於運用到對稱式加密，需要編碼者特定的金鑰，方可解碼，因此可以確保QR Code的來源。
    可用於藥罐上，確保藥品履歷不遭受偽造，另外消費者往往不需知道藥品過於細部的資訊，因此在資料呈現的上，透過資料結構的方式，提供消費者精簡型的藥品電子履歷，當然若想細部查看，仍可透過網址連結查看。

4.結構化QR Code
    將結構化的資料與格式化的資料進行整理，可以針對特定的運用情境，作客製化的呈現。像是將內容、標記符號、座標等不同類型的參數資料，透過特定符號，組合成一組結構化的資訊串列，編碼於QR Code中。當解碼後，系統會再依分割符號拆成各種片段字串，做各別的運用，或與系統存的圖片做結合，呈現給使用者圖文並茂的資訊。
    可用百貨公司、停車場等，告知使用者地點與其他資訊。也可用於大醫院，醫生看診後會開立各種單據，當病患拿到時，往往會發生不知先後的流程，以及路線等問題。透過結構化QR Code，則病患可以使用行動裝置掃描單據上的QR Code，得知所需的資訊，依據附有編號順序的標記符號，知道帶著健保卡到醫學大樓抽血櫃台抽血，去放射科照X光，然後去繳費櫃檯付費，最後再去藥局領藥。

5.資料庫QR Code
    將欲傳遞的各項訊息、圖片位址，儲存於資料庫，賦予每項欄位一組對應的流水序號，並將流水號編碼成QR Code。解碼時依流水序號取得資訊，呈現給使用者。跟上述的結構化QR Code差在訊息的儲存，需使用系統的記憶體空間。
